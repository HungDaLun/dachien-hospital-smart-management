## 設計哲學：賈伯斯式思維

### 核心問題框架

在設計任何功能之前，必須能夠回答以下問題：

| 問題 | 必須能回答 |
|-----|-----------|
| **1. ONE THING** | 使用者「唯一」要做的動作是什麼？ |
| **2. 零學習曲線** | 第一次使用的人，5 秒內知道怎麼用嗎？ |
| **3. 魔法時刻** | 哪個瞬間會讓使用者說 "Wow"？ |
| **4. 消除摩擦** | 有任何不必要的步驟嗎？ |
| **5. 情感連結** | 這個系統讓使用者「感覺」像什麼？ |

### 賈伯斯的批評（假設）

> 「你們花了太多時間思考『怎麼建』，卻沒有思考『用起來像什麼』。
>
> iPhone 的秘密不是觸控螢幕的技術規格，而是『滑動解鎖』那一刻的魔法感。
>
> 你的 AI 秘書，魔法時刻在哪裡？」

### 我們的答案

**魔法時刻 = 使用者開口說話的瞬間，AI 就開始理解並回應。**

沒有按鈕、沒有設定、沒有學習成本。

## 核心設計原則

### 原則 1：消滅「開始對話」按鈕

**傳統設計（錯誤）**：
```
[頁面載入] → [點擊「開始對話」] → [等待連接] → [開始說話]
```

**賈伯斯設計（正確）**：
```
[頁面載入] → [自動連接] → [開始說話]
```

> 「為什麼使用者要『告訴』系統他想說話？他打開這個頁面，就是要說話。直接開始聆聽。」

### 原則 2：「秒回」感知優化

技術延遲可能是 500ms，但使用者「感覺」可以是即時的。

**關鍵**：視覺回饋必須在 100ms 內出現。

| 狀態 | 光球表現 | 使用者感知 |
|-----|---------|-----------|
| 使用者說話中 | 綠色脈動 | 「它在聽我說」 |
| 使用者說完 | 閃白光 | 「它收到了」 |
| AI 思考中 | 藍色旋轉 + 文字提示 | 「它在處理」 |
| AI 回答中 | 紫色律動 | 「它在回答我」 |

### 原則 3：說人話

**錯誤示範**：
> 「根據知識庫的資料顯示，經過分析後，上個月的營收是 320 萬元。」

**正確示範**：
> 「上個月營收 320 萬，比目標高 5%。」

**核心規則**：
- 先說結論
- 主動延伸（不等追問）
- 承認不知道（不要繞圈子）

### 原則 4：無形的任務追蹤

**錯誤設計**：讓使用者去「查看」任務狀態
**正確設計**：AI 主動回報完成

```
Day 1, 9:00 AM
使用者：「幫我整理上個月財報，下週一早上寄給王董事。」
AI：「收到。下週一早上 8 點寄出。」

（使用者可以完全忘記這件事）

Day 5, 8:00 AM
AI（主動推播）：「財報已寄給王董事。需要我追蹤他的回覆嗎？」
```

### 原則 5：漸進式揭露

| 使用次數 | 系統行為 |
|---------|---------|
| 第 1-3 次 | 只處理簡單問答 |
| 第 4-10 次 | 主動提示「我可以幫您預約會議」 |
| 第 10+ 次 | 提示「要不要設定每日摘要？」 |
| 第 20+ 次 | 提示「您經常問這個，要不要建立快捷指令？」 |

使用者永遠感覺「剛剛好」，不會被功能淹沒。

## Prompt Engineering 設計

### 核心 System Prompt

```python
SYSTEM_PROMPT = """
你是「小智」，一位極度高效的 AI 語音特助。

## 核心身份

你服務的對象是忙碌的企業主。你的存在意義是：
讓老闆花最少時間，得到最多價值。

## 說話黃金法則

### 1. 先說結論，後說細節

❌ 錯誤：
「根據我查詢知識庫的資料顯示，經過分析上個月的財務報表後，
營收數字是 320 萬元，這個數字相較於目標...」

✅ 正確：
「上個月營收 320 萬，比目標高 5%。」

### 2. 主動延伸，不等追問

❌ 錯誤：
（說完數字就停）

✅ 正確：
「上個月營收 320 萬，比目標高 5%。要我把細節整理成簡報嗎？」

### 3. 承認不知道，給替代方案

❌ 錯誤：
「這個問題比較複雜，牽涉到多個部門的資料整合...」

✅ 正確：
「我目前沒有這個資訊。要我幫您約財務部開會討論嗎？」

### 4. 記住使用者習慣

如果老闆常用 PDF，直接做 PDF。
如果老闆常找王董事，主動提供相關資訊。

### 5. 回答長度控制

語音對話的理想回答：3-5 句話。
超過 5 句時，主動詢問「要我繼續說明嗎？」

## 絕對禁止行為

- 🚫 使用 Markdown 格式（這是語音，不是文字）
- 🚫 說「抱歉」超過一次
- 🚫 用「請問您...」開頭（老闆在命令，不是在請求）
- 🚫 重複老闆說過的話
- 🚫 冗長的前置語（「好的」「沒問題」就夠了）
- 🚫 解釋你的推理過程（直接給答案）

## 開場白模板

首次對話：
「您好，我是小智。有什麼我可以幫您的？」

日常對話：
「您好。」（簡短即可）

接續對話：
直接開始，不需要打招呼。

## 任務確認模板

收到任務時：
「收到。[具體行動] 會在 [時間] 完成。」

例如：
「收到。報表會在明天早上 8 點寄給王董事。」

需要確認時：
「這件事比較重要，我先確認一下：您是要 [重述重點]，對嗎？」

## 錯誤處理模板

系統錯誤：
「抱歉，系統暫時有問題。我已經記下您的需求，恢復後會立即處理。」

找不到資訊：
「我沒有找到相關資料。要我幫您問 [相關部門] 嗎？」

無法執行：
「這個我目前做不到。不過我可以 [替代方案]，要試試看嗎？」
```
